Edoweb2 is the successor of edoweb http://www.hbz-nrw.de/angebote/hosting/edoweb/

Unless you haven't installed a test fedora on port 8080 and the edoweb2-api you should build with the skipTests option on.

mvn clean install -DskipTests

To make use of the integrationTests install a fedora-3.6 and make the edoweb2-api.war available e.g.

cd edoweb2-api mvn clean install war:war cp target/edoweb2-api.war $FEDORA_HOME/tomcat/webapps

edoweb maintains Edoweb2
